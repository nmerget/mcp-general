## Why MCP General?

<CardGrid stagger>
  <Card title="Config-driven" icon="setting">
    Define tools, resources, and prompts in a single config file — no code
    required.
  </Card>
  <Card title="URL & local files" icon="document">
    Fetch content from remote URLs or read local files. String values are
    auto-detected.
  </Card>
  <Card title="Namespaced" icon="list-format">
    Organize entries into namespaces. Each tool is prefixed with its namespace
    for clarity.
  </Card>
  <Card title="Hot reload" icon="rocket">
    The server watches your config file and re-registers tools, resources, and
    prompts on change.
  </Card>
</CardGrid>
